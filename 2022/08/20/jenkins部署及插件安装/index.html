    jenkins部署

    1、jenkins.war下载

    其实部署很简单，但因为这次升级插件导致全面崩溃。各种插件依赖。主要是都要升级到最新版本还要把java8升级到java11.但因为公司环境都是java8。部署好了都放弃了。
    下载2.289.3是有国内可固定更新的。测试过各版本，建议用这个版本。

    wget https://get.jenkins.io/war-stable/2.289.3/jenkins.war
    2、下载tomcat9

    下载、解压、把webapps目录下的全删除掉。

    wget https://dlcdn.apache.org/tomcat/tomcat-9/v9.0.65/bin/apache-tomcat-9.0.65.tar.gz
tar xzf apache-tomcat-9.0.65.tar.gz
mv apache-tomcat-9.0.65 tomcat9
cd tomcat9/webapps
rm -rf ./*
    3、安装java环境，启动tomcat

    下载java8/maven, 解压。配置环境变量，添加到etc/proifle文件最后

    mv jdk-8u171-linux-x64.tar.gz /opt/
cd /opt && tar -xzf jdk-8u171-linux-x64.tar.gz
mv ~/apache-maven-3.0.5-bin.tar.gz /opt/
tar -xzf apache-maven-3.0.5-bin.tar.gz
vim /etc/profile 
export JAVA_HOME=/opt/jdk1.8.0_171
export JRE_HOME=/opt/jdk1.8.0_171/jre
export M2_HOME=/opt/mvn3.0
export MAVEN_HOME=/opt/mvn3.0
export PATH=$PATH:$JAVA_HOME/bin:$JRE_HOME/bin:$MAVEN_HOME/bin:$NODE_HOME/bin:$SCANNER_HOME/bin

#保存退出
source /etc/profile
#查看 java

    把jenkins.war放到tomcat9的webapps目录下

    mv ~/jenkiins.war ~/tomcat9/webapps/ROOT.war
cd ~/tomcat9/
./bin/catalina.sh start
    4、打开jenkinsweb登陆。

    输入下面目录里的密码登陆，插件全不用装。
    cat /root/.jenkins/secrets/initialAdminPassword

    jenkins sonarqube集成

    1、sonarqube配置

    打开web http://192.168.2.237:9000/ 登陆
    在管理员用户下我的账号、安全、输入jenins生成令牌、记好这个令牌，只出现一次。

    webhook用于jenkins回调sonar扫描结果，好做判断。
    http://192.168.2.237:9000/admin/webhooks
    在webhook页面，创建、输入jenkinshook、url中输入http://192.168.2.20:8080/sonarqube-webhook 、创建。其中192.168.2.20为jenkinsIP

    2、jenkins配置

    下载sonar-scanner4.7 解压到/opt目录下
    登陆jenkins、在系统管理、全局管理、全局配置工具、在SonarQube Scanner把sonar-scanner位置填输入
    http://192.168.2.20:8080/
    name SonarQube Scanner
    SONAR_RUNNER_HOME /opt/sonar-scanner4.7
    在系统管理、凭证管理(credentials Manger)、全域、添加凭证、类型选择secert text
    Secret 输入sonar生成的令牌。
    ID jenkinssonar
    描述 sonarjenkins
    在系统管理、系统配置、SonarQube servers、勾选Environment variables、填入
    Name sonar
    Server URL http://192.168.2.237:9000
    Server authentication token 选择sonarjenkins

    3、pipeline中使用sonarqube

    在pipeline中加入一个stage。 waitForQualityGate是jenkins回调sonar结果。是根据withSonarQubeEnv返回的id调的。增加 sleep(10)更有保障。因为生成可回调结果可能需要时间。

            stage('SonarQube') {
            tools {
                jdk "jdk1.8" // the name you have given the JDK installation in Global Tool Configuration
            }
            environment {
                scannerHome = tool 'SonarQube Scanner' // the name you have given the Sonar Scanner (in Global Tool Configuration)
            }
            steps {
                withSonarQubeEnv(installationName: 'sonar') {
                    sh '''${scannerHome}/bin/sonar-scanner \\
                        -Dsonar.projectKey=${scan_project} \\
                        -Dsonar.sources=. \\
                        -Dsonar.host.url=http://192.168.2.237:9000 \\
                        -Dsonar.login=58ec79f94029eb51b42ec5664bceffdc809fae1e  \\
                        -Dsonar.branch.name=dev \\
                        -Dsonar.java.binaries=target \\
                        -Dsonar.language=java \\
                        -Dsonar.inclusions=**/src/main/java/** \\
                        -Dsonar.qualitygate.wait=true'''
                }
                timeout(time: 1, unit: 'MINUTES') {
                    script {
                        def qg = waitForQualityGate()
                        sh "echo $qg"
                        if (qg.status != 'OK') {
                            error "代码质量分析无法通过，请查看 sonarqube  ${qg.status}"
                        }
                    }
                }			
            }
        }

    或者

            stage('SonarQube analysis') {
            withSonarQubeEnv('sonar') {
                sh '''${scannerHome}/bin/sonar-scanner \\
                -Dsonar.projectKey=${scan_project} \\
                -Dsonar.sources=. \\
                -Dsonar.host.url=http://192.168.2.237:9000 \\
                -Dsonar.login=58ec79f94029eb51b42ec5664bceffdc809fae1e  \\
                -Dsonar.branch.name=dev \\
                -Dsonar.java.binaries=target \\
                -Dsonar.language=java \\
                -Dsonar.inclusions=**/src/main/java/** \\
                -Dsonar.qualitygate.wait=true'''
            } // submitted SonarQube taskId is automatically attached to the pipeline context
        }
        
        // No need to occupy a node
        stage("Quality Gate"){
            timeout(time: 1, unit: 'HOURS') { // Just in case something goes wrong, pipeline will be killed after a timeout
                def qg = waitForQualityGate() // Reuse taskId previously collected by withSonarQubeEnv
                if (qg.status != 'OK') {
                error "Pipeline aborted due to quality gate failure: ${qg.status}"
                }
            }
        }

    完成后即可回调成功。
