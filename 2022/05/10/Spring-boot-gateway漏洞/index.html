    Spring boot gateway漏洞解决

    基本就是建一个挖矿进程和一个守护进程

    /tmp/kdevtmpfsi(挖矿进程) /etc/kinsing(守护进程)

    systemctl status 挖矿进程ID 可查到守护进程还有blade-gateway.jar 就是靠这个进来的。也成了守护进程。

    处理掉这些进程

    我杀过好几次，过段时间就又起来了。包括进程和守护进程。应该是靠gateway远程执行下载进来的。

    所以只好写个脚本一直把这些进程和文件。保证我操作还有资源使用。这个木马有点傻固执，一直是这个名字。

    vim kill_kinsing.sh
#!/bin/bash
function _pid(){
         pid=`ps x|grep kdevtmpfsi|grep -v grep |awk '{print $1}'`
 }

while true
do
        _pid
        if [ -n $pid ];then
                kill $pid && rm -f /tmp/kdevtmpfsi
                pkill kinsing && rm -f /etc/kinsing
                sleep 120
        fi
        sleep 20
done

nohup ./kill_kinsing.sh 2>&1 &

    修复漏洞

    漏洞成因参考：

    https://www.freebuf.com/column/234266.html

    https://sns.bladex.vip/article-15004.html

    lsof -p gateway的进程ID 可看到gateway打开的文件和连接的端口，

    scanner-09.ch1.censys-scanner.com:59200

    scanner-08.ch1.censys-scanner.com:57238

    这两货就很可疑了。对比下正常的gateway，没有这两货就是它了。查下解析的IP也知道是它。

    直接在nginx 反向代理gateway那段server配置加上

    if ($request_uri ~ "/actuator") {
   return 403;
}

    漏洞是gateway中有actuator模块可执行注册写入的权限。

    在nginx反向代理时匹配到actuator让它返回错误。
