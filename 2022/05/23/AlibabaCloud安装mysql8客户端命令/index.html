    AlibabaCloud安装mysql8客户端命令

    中间件 部署安装 mysql mysql-cli
    发布时间 : 2022-05-23 16:40
    阅读 :
    1. 直接下载安装会报错,需要安装GLIBC2.28版本
    2. 确认make 版本在4.2及GCC等devtoolset-8-gcc
    3. 安装make及开发工具新版本：
    4. 开始安装mysql-client

    直接下载安装会报错,需要安装GLIBC2.28版本

    提示需要GLIBC_2.25和2.28版本，目前的版本是2.17

    [root@monitoring rpm-gpg]# ldd --version
ldd (GNU libc) 2.17
Copyright (C) 2012 Free Software Foundation, Inc.
This is free software; see the source for copying conditions.  There is NO
warranty; not even for MERCHANTABILITY or FITNESS FOR A PARTICULAR PURPOSE.
Written by Roland McGrath and Ulrich Drepper.

    这个GLIBC没装好，系统命令都用不了。特别要注意. 这个是链接过来的。要保证原文件 libc-2.17.so暂时不要删除

    [root@monitoring rpm-gpg]# ls -l /lib64/libc.so.6
    lrwxrwxrwx 1 root root 12 May 19 09:39 /lib64/libc.so.6 -> libc-2.17.so

    如果不能用报错 error while loading shared libraries: libc.so.6: cannot open shared object file: No such file or directory

    export LD_PRELOAD=”/lib64/libc-2.17.so”

    确认make 版本在4.2及GCC等devtoolset-8-gcc

    因为安装glibc2.28需要make及开发工具比较新的版本，不然会报下面这个错误: make bison compiler

    checking if gcc -B/usr/bin/ is sufficient to build libc... no
checking for nm... nm
checking for python3... python3
configure: error: 
*** These critical programs are missing or too old: make bison compiler
*** Check the INSTALL file for required versions.

    安装make及开发工具新版本：

    centos 安装命令

    yum install centos-release-scl
yum install devtoolset-8-gcc devtoolset-8-gcc-c++ devtoolset-8-binutils
scl enable devtoolset-8 bash
echo "source /opt/rh/devtoolset-8/enable" >> /etc/profile

    Alibaba Cloud LinuxOS需要下面的安装命令

    yum install -y alinux-release-experimentals
yum install -y devtoolset-8-gcc devtoolset-8-gdb devtoolset-8-binutils devtoolset-8-make
scl enable devtoolset-8 bash
echo "source /opt/rh/devtoolset-8/enable" >> /etc/profile

    下载安装make4.2

    wget http://ftp.gnu.org/gnu/make/make-4.2.tar.gz
tar xzf make-4.2.tar.gz
cd make-4.2
make 
make install
rm -f /usr/bin/make
cp ./make /usr/bin/make

    先下载2.28包，解压，创建build目录，进入build目录, 配置编译环境，make里要很久. 有的情况在后面make install 有ld: cannot find -lnss_test2 错误。这个错误可以忽略。

    各个版本的glibc下载地址：https://mirrors.sjtug.sjtu.edu.cn/gnu/glibc/

    wget https://mirrors.sjtug.sjtu.edu.cn/gnu/glibc/glibc-2.28.tar.gz
tar xzf glibc-2.28.tar.gz 
cd glibc-2.28/
mkdir build 
cd build
../configure --prefix=/usr --disable-profile --enable-add-ons --with-headers=/usr/include --with-binutils=/usr/bin
make -j4
make install 
if test -r /usr/include/gnu/stubs-64.h && cmp -s /opt/glibc-2.28/build/stubs.h /usr/include/gnu/stubs-64.h; \
then echo 'stubs.h unchanged'; \
else /usr/bin/install -c -m 644 /opt/glibc-2.28/build/stubs.h /usr/include/gnu/stubs-64.h; fi
rm -f /opt/glibc-2.28/build/stubs.h
/opt/glibc-2.28/build/elf/sln /opt/glibc-2.28/build/elf/symlink.list
rm -f /opt/glibc-2.28/build/elf/symlink.list
test ! -x /opt/glibc-2.28/build/elf/ldconfig || LC_ALL=C \
  /opt/glibc-2.28/build/elf/ldconfig  \
                        /lib64 /usr/lib64
LD_SO=ld-linux-x86-64.so.2 CC="gcc -B/usr/bin/" /usr/bin/perl scripts/test-installation.pl /opt/glibc-2.28/build/
/usr/bin/ld: cannot find -lnss_test2
collect2: error: ld returned 1 exit status
Execution of gcc -B/usr/bin/ failed!

Please read the FAQ and the README file and check the following:
- Did you change the gcc specs file (necessary after upgrading from

    开始安装mysql-client

    #mysql就是client, mysql-server才是服务器. 按y确认安装。

    wget https://dev.mysql.com/get/mysql80-community-release-el7-6.noarch.rpm

yum localinstall mysql80-community-release-el7-6.noarch.rpm

cd /etc/pki/rpm-gpg
wget https://repo.mysql.com/RPM-GPG-KEY-mysql-2022
rpm --import /etc/pki/rpm-gpg/RPM-GPG-KEY-mysql-2022
