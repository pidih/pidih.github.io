    Elasticsearch7.16.1和kibana集群部署

    日志 elasticsearch;elk;es
    发布时间 : 2022-09-04 15:45
    阅读 :
    1. 一、概述
    2. 二、部署elasticsearch
    3. 三、kibana部署

    一、概述

    本次使用es版本为7.16.3, kibana版本也是7.16.3

    好像7.10版本就自带jdk了。值得注意的是，elasticsearch不支持root用户启动，自己需创建一个普通用户启动。

    master: 192.168.2.232

    node: 192.168.2.231, 192.168.2.233

    二、部署elasticsearch

    下载es7.16.3版本

    wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-7.16.3-linux-x86_64.tar.gz
tar -xzf elasticsearch-7.16.3-linux-x86_64.tar.gz
mv elasticsearch-7.16.3/ elasticsearch7.16

    master节点配置, master节点比node节点多了cluster.initial_master_nodes参考，意思是集群初始化主节点

cluster.initial_master_nodes: ["able231","able232","able233"]

    xpack ssl

    bin/elasticsearch-certutil --help
    csr - generate certificate signing requests
    cert - generate X.509 certificates and keys
    ca - generate a new local certificate authority

bin/elasticsearch-certutil ca //创建集群节点安全认证机构
bin/elasticsearch-certutil cert --ca elastic-stack-ca.p12  //为节点颁发证书

//如果在创建证书期间设置密码需要把认证密码存储器起来，方便es节点认证
bin/elasticsearch-keystore add xpack.security.transport.ssl.keystore.secure_password
bin/elasticsearch-keystore add xpack.security.transport.ssl.truststore.secure_password
mkdir config/certs
mv elastic-* config/certs

    #jvm.options

    -Xms8g  //memory half
-Xmx8g

10-:-XX:+UseG1GC //openjdk10+
10-:-XX:G1ReservePercent=25
10-:-XX:InitiatingHeapOccupancyPercent=30

    证书配置文件位置 xpack.security.transport.ssl.keystore.path: certs/elastic-certificates.p12 是配置文件目录下的certs， 也就是/opt/elasticsearch7.16/config/certs/ 目录下

    把证书拷备到另外两台相同的位置。

    证书放后就可以启动三台es了。注意最先启动master节点。这样方便观察加入状态

    ./bin/elasticsearch -d -p pid  
pkill -F pid

    查看日志节点加入情况

    tail -n 40 -f logs/able-es.log

    看节点都加入集群后，配置账号密码。按y确认生成密码

    bin/elasticsearch-setup-passwords auto -u "http://192.168.2.232:9200"  //自动设置内置账户密码

    三、kibana部署

    下载

    wget https://artifacts.elastic.co/downloads/kibana/kibana-7.16.3-linux-x86_64.tar.gz
tar xzf kibana-7.16.3-linux-x86_64.tar.gz
mv kibana-7.16.3 kibana

    修改配置文件

    server.port: 5601
server.host: "192.168.2.232"
server.name: "kibana-able-acdata"
pid.file: ./kibana.pid

elasticsearch.hosts: ["http://192.168.2.231:9200","http://192.168.2.232:9200","http://192.168.2.233:9200"]
#elasticsearch.username: "elastic"
#elasticsearch.password: "pass"

kibana.defaultAppId: "home"
i18n.locale: "en"

xpack.security.encryptionKey: "386400e4862228d0683bc6df52c570c6"
#various ui
xpack.apm.enabled: false
xpack.infra.enabled: false
xpack.ml.enabled: false
xpack.reporting.enabled: false
xpack.graph.enabled: false

    启动kibana， kibana-keystore可以创建账号密码记在里面。配置文件中就可以注释掉elasticsearch.username和elasticsearch.password

    这样配置文件中就没有明文密码记录

    ./bin/kibana-keystore list
./bin/kibana-keystore create
./bin/kibana-keystore add elasticsearch.username
./bin/kibana-keystore add elasticsearch.password
./bin/kibana-keystore list

./bin/kibana  #启动kibana

    浏览器打开 http://192.168.2.232:5601/ 输入账号密码就可登陆。
