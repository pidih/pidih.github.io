    log.io部署与使用

    日志 日志 log.io 实时日志
    发布时间 : 2023-09-23 19:28
    阅读 :
    1. 一、环境说明
    2. 二、部署Log.io
      1. 1、安装nodejs
      2. 2、安装 log.io
      3. 3、配置服务端
      4. 4、配置日志采集端
      5. 5、运行
      6. 6、WEB展示

    一、环境说明

    1、操作系统为ubuntu20.04

    2、此log.io依赖nodejs，需要先部署好nodejs.

    3、日志由log.io-file-input采集然后输出到log.io的6689端口。用户就可以在6688端口上查看内容了。

    4、https://github.com/NarrativeScience-old/log.io 官方链接

    二、部署Log.io

    1、安装nodejs
    sudo apt update -y
sudo apt install nodejs
sudo apt install npm
    2、安装 log.io
    sudo npm install -g log.io
sudo npm install -g log.io-file-input
    3、配置服务端

    千万别写错了，我把server.json写成了server.conf 后面找了两三小时原因。如果没有这个配置也是可以运行的，只是监听的端口是127.0.0.1不太方便，也不太方便WEB访问。我还傻傻的使用NGINX代理转发才实现访问。这样还可以加密访问。用户名密码是”username1”: “password1”

    vim ~/.log.io/server.json
{
  "messageServer": {
    "port": 6689,
    "host": "192.168.2.123"
  },
  "httpServer": {
    "port": 6688,
    "host": "192.168.2.123"
  },
  "debug": false,
  "basicAuth": {
    "realm": "abc123",
    "users": {
      "username1": "password1"
    }
  }
}
    4、配置日志采集端

    这里配置了采集两个日志，achz和4008，path是指定路径。 6689端口是服务器接收端口。如果以后增加采集点可以添加配置文件input中增加一个配置就好，然后kill掉input进程，再启动input

    vim ~/.log.io/inputs/file.json
{
  "messageServer": {
    "host": "192.168.2.123",
    "port": 6689
  },
  "inputs": [
    {
      "source": "achz",
      "stream": "app1",
      "config": {
        "path": "/home/netsin/aciot/achz/target/achz-0.0.1-SNAPSHOT.jar.log"
      }
    },
    {
      "source": "4008",
      "stream": "app2",
      "config": {
        "path": "/root"
      }
    }
  ]
}
    5、运行
    log.io-server &
log.io-file-input &
ps aux|grep log.io
root     1253084  0.2  0.2 694460 65904 pts/2    Sl   14:22   0:05 node /usr/local/bin/log.io-server
root     1255524  0.6  0.1 614788 46520 pts/0    Sl   14:27   0:12 node /usr/local/bin/log.io-file-input
root     1268950  0.0  0.0   8164  2560 pts/1    S+   14:57   0:00 grep --color=auto log.io
    6、WEB展示

    http://192.168.2.123:6688/

    6688端口是WEB展示端口，只要输对了用户和密码就可以访问了。

    右上角有个查找标记的小窗口，可以过滤查找相关内容，回车就可以看到标记。

    7、其他配置

    添加进程管理pm2:

    {
  "name":"log.io",  // 指定明后才能
  "min_uptime": "12h",  // 由于过段时间就无法获取,进行过段时间重启操作
  "script": "/usr/local/node/bin/log.io-file-input",
  "exec_mode": "fork",
  "watch": true,
  "cwd": "/root/.log.io/inputs",  // 指定路径,配置文件
  "max_restarts": 3,
  "restart_delay": 4000,
  "error_file": "./log-io_err.log",
  "out_file": "./log-io_out.log"
}

    存储方面，log.io只监视不储存，所以不会有储存负担。
