    黄建波

    一、部署环境

    系统： Centos7

    3台服务器：192.168.2.195、192.168.2.196、192.168.2.197

    port: 6379 6380

    redis版本： redis-5.0.4.tar.gz

    二、准备工作

    3台服务器都要执行同样操作

    1、关掉防火墙和selinux
    systemctl stop firewalld		#关防火墙
systemctl disable firewalld		#设置开机不启动防火墙
vim /etc/sysconfig/selinux		#SELINUX=disabled 改disabled为关闭 enabled是开启
SELINUX=disabled
    2、设置系统参数优化redis
    vim /etc/sysctl.conf		#在最后面增加两行
vm.overcommit_memory = 1
net.core.somaxconn = 2048

sysctl -p 					# 使上面的内容生效

    echo never > /sys/kernel/mm/transparent_hugepage/enabled
vim /etc/rc.d/rc.local		
if test -f /sys/kernel/mm/transparent_hugepage/enabled; then
   echo never > /sys/kernel/mm/transparent_hugepage/enabled
fi

if test -f /sys/kernel/mm/transparent_hugepage/defrag; then
   echo never > /sys/kernel/mm/transparent_hugepage/defrag
fi

chmod +x /etc/rc.d/rc.local  # 增加可执行权限，开启就能执行

vim /etc/security/limits.conf		#在最后增加下面两行，软件硬件打开文件个数，默认1024太小。
* soft nofile 65535
* hard nofile 65535
    3、下载安装依赖
    yum groupinstall "Development tools"		#安装make gcc gcc-c++ 
wget http://download.redis.io/releases/redis-5.0.4.tar.gz		#下载redis5的安装包
tar -xzf redis-5.0.4.tar.gz -C /opt/redis5

    三、安装redis

    三台服务器都做同样操作

    1、准备主从文件夹
    cd /opt/ && mkdir redis_cluster  #准备集群目录
cd redis_cluster && mkdir 6379redis 6380redis	#主从目录
    2、安装redis

    这是在192.168.2.195 上操作，所以IP就是192.168.2.195。 其他服务器上的IP要改当前服务器的IP。

    cd /opt/redis5
make && make install
#安装好后就有redis-cli redis-server等命令了
vim /opt/redis_cluster/6379redis/redis.conf		#redis配置文件增加以下内容，IP要改为服务器的IP
bind 192.168.2.195
port 6379
daemonize yes
pidfile /var/run/redis_6379.pid
logfile "/opt/redis_cluster/6379redis/redis.log"
appendonly yes
appendfsync always
cluster-enabled yes
cluster-config-file nodes-6379.conf
cluster-node-timeout 15000
requirepass acredis365

vim /opt/redis_cluster/6380redis/redis.conf      #redis从的配置文件。IP要改为服务器的IP
bind 192.168.2.195				#这是监听的IP
port 6380						#监控的端口
daemonize yes					#放在后台运行
pidfile /var/run/redis_6380.pid		#指定PID所在位置
logfile "/opt/redis_cluster/6380redis/redis.log"	#指定日志位置
appendonly yes								#这个是硬盘同步
appendfsync always							#总是同步
cluster-enabled yes							#开户集群模式
cluster-config-file nodes-6380.conf			#集群的配置文件位置，这个配置文件是会自动生成的
cluster-node-timeout 15000					#集群同步的超时时间
requirepass acredis365						#redis认证密码

    到这里redis部署好了，现在启动redis

    cd /opt/redis_cluster/6379redis && redis-server redis.conf
cd /opt/redis_cluster/6380redis && redis-server redis.conf

    将启动写到开机自启文件中

    vim /etc/rc.d/rc.local				#增加到最后行就行
cd /opt/redis_cluster/6379redis && redis-server redis.conf
cd /opt/redis_cluster/6380redis && redis-server redis.conf

    还有个事要注意，登陆时 加入 -a会一直使用这个密码登陆所有集群，但登陆某节点后使用auth登陆的，要查内容在其他节点就查不到，需要认证才能查到。

OK

"world"

    五、集群管理

    1 添加新主节点
    redis-cli --cluster add-node new_host:new_port existing_host:existing_port --cluster-master-id node_id
    2 hash槽重新分配

    添加完新节点后，需要对新添加的主节点进行hash槽重新分配，这样该主节点才能存储数据，redis共有16384个槽。

    redis-cli --cluster reshard host:port --cluster-from node_id --cluster-to node_id --cluster-slots <args> --cluster-yes
    3添加新从节点
    redis-cli --cluster add-node new_host:new_port existing_host:existing_port --cluster-slave --cluster-master-id node_id
    4 删除节点
    redis-cli --cluster  del-node host:port node_id
